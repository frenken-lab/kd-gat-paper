# Figure Authoring

Interactive figures are self-contained Svelte apps built with [SveltePlot 0.12](https://svelteplot.dev) and bundled into single HTML files via `vite-plugin-singlefile`.

## Figure Structure

Each figure lives in `interactive/src/figures/<name>/` with at minimum:

```
interactive/src/figures/umap/
  App.svelte    # Renderer — SveltePlot marks only, no data transforms
  data.json     # Pre-computed data (from KD-GAT export pipeline)
```

`index.html` and `main.js` are auto-generated by `build.js` if missing. You only need to create `App.svelte` (and optionally `data.json`).

## Creating a New Figure

1. Create the directory:
   ```bash
   mkdir interactive/src/figures/my-figure
   ```

2. Add `data.json` — exported from the KD-GAT pipeline via `export_paper_data.py`. The data must be fully pre-computed (no transforms in the figure).

3. Create `App.svelte`:
   ```svelte
   <script>
     import data from './data.json'
     import { Cell, Dot, Line } from 'svelteplot'
   </script>

   {#if data.length > 0}
     <Cell>
       <Dot {data} x="x" y="y" fill="category" />
     </Cell>
   {:else}
     <p>Awaiting data export</p>
   {/if}
   ```

4. Test it:
   ```bash
   cd interactive
   FIGURE=my-figure npx vite dev
   ```

5. Build all figures:
   ```bash
   make figures
   ```
   Output: `_build/figures/my-figure.html`

6. Reference in content via iframe:
   ```markdown
   :::{iframe} https://frenken-lab.github.io/kd-gat-paper/assets/html/submission/my-figure.html
   :label: fig-my-figure
   :width: 100%
   Caption text here.
   :::
   ```

## Architecture Diagrams (spec-driven)

Architecture diagrams (architecture, gat, kd-gat, kd-vgae, vgae) use a different workflow: **YAML specs** instead of `data.json`. See the [Diagram Authoring Guide](Diagram-Authoring-Guide.md) for the full spec reference.

## Key Conventions

- **Dumb renderers**: Figures import `data.json` and plot it. All preprocessing (sampling, flattening, ROC computation, layout) happens in `export_paper_data.py` in the KD-GAT repo. No data transforms in `.svelte` files.
- **SveltePlot only**: Use `<Cell>`, `<RectY>`, `<Line>`, `<Dot>`, `<Arrow>` marks. No D3 or other chart libraries.
- **Handle empty data**: Show "Awaiting data export" when `data.json` is `[]` or `{}`. This allows building figures before data is available.
- **No hardcoded colors**: Use role names (`vgae`, `gat`, `kd`) that resolve via the palette system in `styles.yml`.

## Build System Internals

`build.js` orchestrates the build because `vite-plugin-singlefile` requires `inlineDynamicImports`, which is incompatible with multiple Rollup entry points in a single pass. For each figure:

1. Scans `src/figures/` for directories containing `App.svelte`
2. Auto-generates `index.html` + `main.js` from templates if missing
3. Runs `npx vite build` with `FIGURE=<name>` injected
4. Renames output from `_build/figures/index.html` → `_build/figures/<name>.html`

## Styling

`interactive/styles.yml` defines the shared palette and font settings. Two virtual Vite modules expose it at build time:

- `virtual:styles` — imports `styles.yml` as a JS object
- `virtual:theme-vars.css` — generates `:root { ... }` CSS variables from palette/font sections

Figures pick up theme colors automatically through these modules.

## iframe Deployment

Figures require `<script>` execution, which curve.space's SPA strips. They're hosted on GitHub Pages and embedded as iframes in both the curve.space report and TMLR submission. The TMLR export rewrites iframe URLs to relative paths (`assets/html/submission/`) for anonymous self-contained submissions.
